# Ralph Progress Log - 竞品直播间爬取

## Codebase Patterns
<!-- 每次迭代发现的可复用模式追加在此 -->

- Use `sys.path.insert(0, str(Path(__file__).parent.parent))` for relative imports across all modules
- Groq Python SDK (`from groq import Groq`) accepts OpenAI-compatible `.chat.completions.create()` API — same message format works for both MiniMax and Groq
- httpx raised as `HTTPStatusError` on non-2xx responses; use `except httpx.HTTPStatusError as e` then access `e.response` for status code
- No mypy/pyright configured in this project — "Typecheck passes" = valid Python syntax + importable module
- Import groq inside try block or at top-level; groq is already in requirements.txt

- What was implemented: Wrapped MiniMax httpx.post call in try/except; on HTTPStatusError or general Exception, auto-switch to Groq llama-3.3-70b-versatile using same messages list. Both fail → original MiniMax exception re-raised so scheduler logs it.
- Files changed: `analyzer/claude_analyze.py`

  - httpx raises `HTTPStatusError` (not just HTTPError) on non-2xx, caught with `except httpx.HTTPStatusError`
  - Store provider name in a variable so the final log prints which API succeeded
  - Groq key checked lazily at fallback time (not upfront) so MiniMax-only environments still work
  - Douyin room_id can be extracted from URL pathname: `location.pathname.match(/\/live\/(\d+)/)` — more reliable than parsing __NEXT_DATA__ scripts
  - Use `page.route("**/*.m3u8*", handler)` then `await page.wait_for_timeout()` to capture intercepts — route stays active during the wait
  - Always `unroute()` in a try/except finally pattern to avoid leaking route handlers
